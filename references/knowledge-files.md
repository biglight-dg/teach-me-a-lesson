# 지식 파일 조회·저장 가이드

이 스킬은 답하기 전에 **지식 파일**(사용자가 모아둔 마크다운 노트 폴더)에 같은 주제 문서가 있는지 먼저 찾아본다. 있으면 그 비유·내용·출처를 근거로 삼아 설명이 더 정확하고 일관돼진다. **없으면 조용히 Claude 자체 지식으로 설명한다(에러로 멈추지 않는다).** 지식 파일은 **선택 사항**이라, 하나도 없어도 스킬은 그대로 작동한다.

## 지식 파일 폴더 위치

기본값(아무 설정 안 했을 때) 후보를 이 순서로 찾는다. 먼저 존재하는 것을 쓴다.

1. 현재 작업 폴더의 `./knowledge/`
2. `~/.claude/teach-me-a-lesson/knowledge/`

> **내 폴더로 바꾸고 싶으면:** 이미 마크다운 노트를 모아둔 폴더가 있다면 그 경로를 쓰면 된다. 사용자에게 "지식 파일을 두신 폴더가 따로 있나요?"라고 물어 경로를 받거나, 위 기본 후보를 그대로 쓴다. 폴더가 존재하지 않는 건 정상이며 오류가 아니다 — 그냥 자체 지식으로 설명한다.

## 조회 절차 (explain 모드)

1. 위 후보 중 존재하는 지식 폴더를 찾는다. **없으면 → 자체 지식으로 폴백, 끝.**
2. 폴더 안 `.md` 파일들의 파일명(과 있으면 인덱스 파일)에서 질문의 핵심 키워드를 매칭해 관련 문서를 1~2개 고른다. (예: "프롬프트 어떻게 써" → 파일명에 "프롬프트" 포함)
3. 매칭된 문서를 읽어 비유·구조·핵심 교훈을 설명의 근거로 삼는다. 그대로 베끼지 말고 사용자 질문 눈높이에 맞게 재구성한다.
4. 답변 끝에 그 파일을 출처로 적는다. 예: `> 출처: knowledge/프롬프트 잘 쓰는 법.md`
5. 매칭되는 문서가 없으면 자체 지식으로 설명하고 `> 출처: Claude 보유 지식 기반 일반 설명`이라고 밝힌다.

### 효율 팁
- Grep/검색 도구로 폴더 안 키워드를 빠르게 훑어 후보를 좁히면 전체를 다 읽지 않아도 된다.
- 인덱스 파일(JSON 등)이 있으면 그걸 먼저 읽어 후보를 좁힌다. 인덱스가 없으면 파일명만으로 충분하다.

## 저장 절차 (knowledge 모드, `/tl-knowledge`)

배운 개념을 나중에 다시 찾아볼 수 있도록 **마크다운 노트**로 남긴다.

1. 저장 폴더는 위 "지식 파일 폴더 위치" 규칙을 따른다. 없으면 첫 번째 후보(`./knowledge/`)를 만들어 쓴다.
2. 파일명: 주제를 알아보기 쉽게. 예: `knowledge/MCP가 뭔지.md` (날짜 접두사를 원하면 `YYYYMMDD_제목.md`).
3. 노트는 explain 모드와 같은 구조로 적되 **간결하게**: 한 줄 핵심 → 🔑 비유 → 💡 핵심 교훈 3가지 → (필요 시) 🧭 실전 한 걸음 → 출처.
4. 저장 후 사용자에게 **저장 위치(경로)** 를 알려준다.

> 형식을 통일해두면 나중에 explain 모드가 이 노트를 다시 근거로 활용한다. 즉, 쓰면 쓸수록 자기만의 지식베이스가 쌓이는 구조다.

## (선택) 인덱스 파일 연결

이미 인덱스를 쓰는 환경이라면 아래 같은 형식이면 자동 연결된다(필수 아님):

```json
{
  "items": [
    { "title": "MCP 쉽게 이해하기", "path": "knowledge/MCP가 뭔지.md", "tags": ["MCP", "입문"] }
  ]
}
```

인덱스의 `path`가 상대경로면 인덱스 파일이 있는 폴더 기준으로 해석한다. 인덱스가 없으면 이 절은 무시하고 파일명 매칭만 쓴다.
